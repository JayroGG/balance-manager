# balance-mobile

Expo (React Native) client for the [`balance`](../../Node/Projects/balance) personal-finance REST API.
Record incomings/expenses, organize savings into **vaults**, and see your **total / available /
per-vault** balance. Thin client — the backend owns all business logic and the money boundary.

> Stack & decisions: see `CLAUDE.md`, `ARCHITECTURE.md` (diagrams), `PRD.md` (contract), and
> `.claude/ADR/`. TL;DR: Expo SDK 54 (managed), expo-router, Redux Toolkit + RTK Query, JS/JSX.

## Prerequisites

- Node 20+, npm
- **Expo Go** app on your phone (iOS/Android) — the prototype runs in Expo Go (no dev build needed)
- The `balance` backend running locally (see below)

## Setup

```bash
npm install
```

Create the per-environment files (not committed — see `.gitignore`):

```bash
# .env.dev
API_URL=http://<your-LAN-ip>:3000   # backend host:port; NO /api prefix
AUTH_BYPASS=true                     # prototype: skip login, send placeholder token
APP_ENV=dev
```

Also create `.env.stage` and `.env.prod` (same keys; `AUTH_BYPASS` must be `false` outside dev/stage).
Use your machine's **LAN IP**, not `localhost`, so a physical phone can reach the backend.

## Run

```bash
APP_ENV=dev   npx expo start    # default; scan the QR with Expo Go
APP_ENV=stage npx expo start
APP_ENV=prod  npx expo start
```

Start the backend in another terminal:

```bash
cd ../../Node/Projects/balance
NODE_ENV=stage npm run migrate && NODE_ENV=stage npm run seed   # first time
NODE_ENV=stage npm start
```

## Health checks

```bash
npx expo-doctor
npx expo-modules-autolinking verify -v   # should list only Expo-SDK native modules (Expo-Go safe)
npm test
```

## Builds (later — leaves Expo Go)

The MVP needs no native build. When a native module is added (MMKV, Auth0, …), switch to a dev build:

```bash
npx expo prebuild                       # generates android/ios (not committed)
eas build --profile development         # dev client; see eas.json (development/preview/production)
```

## Environment / build notes

- Config flows `.env.${APP_ENV}` → `app.config.js` (`extra`) → `expo-constants` → `src/utils/config.js`.
  We do **not** use `react-native-config` (managed workflow — ADR-003).
- `android/` and `ios/` are intentionally **not** committed; they're regenerated by `expo prebuild`.
- Cached data is persisted (redux-persist) for fast cold starts and offline reads; the auth token lives
  in `expo-secure-store`, never in AsyncStorage (ADR-006/007).

## Endpoint summary (backend)

`GET /balance` · `/transactions` (CRUD, filters) · `/categories` (CRUD) · `/vaults` (CRUD +
`/:id/history`, `/:id/allocate`, `/:id/withdraw`). Money is **decimal**; soft-deleted resources return
`404`. Full contract in `PRD.md` §4.
